# How to Contribute to M3 Framework

Thank you for your interest in improving the M3 Framework!
We believe that a pragmatic security standard for the "Fractional Economy" must be community-driven to keep pace with rapid changes in AI and regulations (EU AI Act, GDPR, etc.).

This document outlines the process for proposing changes, adding new tools to the matrix, or refining the core methodology.

## 🚀 What We Are Looking For

We welcome contributions in the following areas:

1.  **Regulatory Updates:** Corrections or additions based on new legal texts (e.g., changes in EU AI Act articles).
2.  **Tooling Matrix:** Adding accessible, low-friction tools for the *Mount* or *Monitor* phases.
    * *Note:* Please avoid vendor marketing. Suggest tools that you have personally tested.
3.  **Clarifications:** Improvements to wording that make the framework easier to understand for non-technical founders.
4.  **Templates:** New policy templates (e.g., specific prompts for HR, Devs, Marketing).
5.  **Translations:** Helping localize the framework into other languages.

## 🚫 What We Do Not Merge

* **Vendor Spam:** Purely promotional content without practical value.
* **Complexity:** Changes that introduce heavy bureaucracy (e.g., mandatory 50-page risk assessments) which contradict the M3 "Action-First" philosophy.
* **Theoretical Fluff:** Academic concepts that cannot be applied by a team of 10 people.

## 📝 The Workflow

We follow the standard GitHub "Fork & Pull" workflow:

1.  **Fork the Repository:** Click the "Fork" button at the top right of this page.
2.  **Create a Branch:** Create a new branch for your specific change.
    * Format: `fix/typo-in-intro` or `feature/new-dlp-section`.
3.  **Make Changes:** Edit the Markdown (`.md`) files. Please keep formatting clean.
4.  **Commit:** Write a clear, concise commit message explaining *why* you made the change.
5.  **Submit a Pull Request (PR):**
    * Go to the "Pull Requests" tab in the original repository.
    * Click "New Pull Request".
    * Compare across forks and select your branch.
    * **Important:** In the PR description, explain how your change helps SMBs/Fractional experts.
